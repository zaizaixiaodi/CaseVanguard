# Skill：交付物生成

## 1. 案件复杂度判断

根据以下指标综合判断：

| 指标 | 简单 | 中等 | 复杂 |
|------|------|------|------|
| 证据数量 | ≤5 份 | 6-10 份 | >10 份 |
| 涉及主体 | ≤2 个 | 3-5 个 | >5 个 |
| 争点数量 | 1-2 个 | 3-4 个 | >4 个 |
| 法律关系 | 单一法律关系 | 2-3 种法律关系 | 多种法律关系交叉 |

**判断规则：** 取各指标的最高级别作为案件复杂度。如有争议，优先取较高级别。

**本案类型映射：**
- 建设工程施工合同纠纷 + 多方主体 → 默认"复杂"
- 民间借贷 → 默认"简单"
- 一般合同纠纷 → 默认"中等"

## 2. 字数分配

| 案件复杂度 | 正文字数 | Executive Summary | 关键细节速查表 |
|-----------|---------|-------------------|---------------|
| 简单 | 1000-1500 字 | 不需要 | 可选 |
| 中等 | 1500-2000 字 | 不需要 | 建议包含 |
| 复杂 | 2000-3000 字 | 必须，≤200 字 | 必需 |

**各章节字数分配（复杂案件）：**

| 章节 | 目标字数 | 说明 |
|------|---------|------|
| Executive Summary | ≤200 字 | 简明摘要，下游智能体可据此建立基本认知 |
| 一、案件概况 | ~300 字 | 谁、什么事、纠纷性质、原告主张；仅记载有证据支撑的客观事实 |
| 二、案件大事记 | ~400 字 | 时间线表格（日期+事件+证据来源） |
| 三、初步证据目录 | ~500 字 | 按组分类，每组概述+关键发现+警告 |
| 四、风险提示与证据链断裂点 | ~400 字 | 证据缺口、不利事实、陈述与证据的冲突 |
| 五、下一步行动计划 | ~400 字 | 当事人配合事项 + 律师推进事项 |
| 附：关键细节速查表 | 不计入正文字数 | 每条1-2句话+原文锚点 |

## 3. Executive Summary 撰写规则

**适用条件：** 仅复杂案件必须生成。

**内容要求：**
- ≤200 字
- 纯事实摘要，不含法律分析判断
- 包含：当事人、案由、核心金额、关键风险点
- 不使用"可能"、"建议"等分析性表述
- 面向下游智能体：阅读此段后即可建立案件基本认知

**示例：**
> 苏茂公司（国企）与武汉成丰学校签订1.55亿元建设工程施工合同，工程已完工交付使用但仅付款2,000万元（不足15%），尚欠约1.35亿元。付款协议引入9家关联主体但均未盖章。银行流水显示学校收入被集团抽走、个人从对公账户取现，构成人格混同。中信银行5,000万融资未用于工程款。存在资产处置风险。

## 4. 各章节撰写规范

### 一、案件概况（~300字）
- 谁告谁、什么事、纠纷性质
- 原告的核心主张（仅记载有证据支撑的部分）
- 核心金额（合同价/已付/欠付）
- 不做法律评价

### 二、案件大事记（~400字）
- 以表格形式呈现关键事件
- 每行：日期 | 事件 | 证据来源
- 仅收录对案件有实质影响的事件
- 来源：`workspace/案件时间线-timeline.md`

### 三、初步证据目录（~500字）
- 按 reading-plan 的分组展示
- 每组：组名 + 证据列表 + 关键发现（1-2句）+ 警告（如有）
- 不展开每份证据的详细内容（详细内容在精要大合集）
- 来源：`workspace/精要大合集-evidence-collection.md`

### 四、风险提示与证据链断裂点（~400字）
- 4.1 证据缺口：哪些要件缺少证据支撑
- 4.2 不利事实：对原告不利的证据内容
- 4.3 陈述与证据的冲突：当事人陈述与证据记载不一致之处
- 来源：`workspace/交叉验证报告-cross-verify-report.md`

### 五、下一步行动计划（~400字）
- 5.1 当事人配合事项：需要当事人补充的证据、确认的事实
- 5.2 律师推进事项：法律检索方向、诉讼策略建议、保全申请
- 来源：cross-verify-report 的"下一步行动建议" + case-context.json 的 focus_points

### 附：关键细节速查表（不计入正文）
- 表格格式：编号 | 细节类型 | 内容 | 来源 | 原文锚点
- 收录所有对案件有决定性影响的细节
- 每条1-2句话
- 必须包含原文锚点（如 `E001.md#L31`）

## 5. 自检规则

生成报告后，执行以下自检：

1. **字数自检：** 统计正文字数（不含标题、表格、元数据），确认在目标范围内
2. **锚点自检：** 速查表每条是否都包含原文锚点
3. **事实来源自检：** 正文中每个事实陈述是否可追溯到证据编号
4. **不做法律判断自检：** 确认无"合同有效"、"被告构成违约"等法律结论性表述
