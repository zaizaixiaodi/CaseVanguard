# /context — 查看/补充案件上下文

> 适用于案件初始化之后的任何阶段。展示案件背景信息，支持律师补充新信息。

## 前置检查

1. 读取 `workspace/meta/case-context.json`。
   - 如果文件不存在 → 提示"尚未初始化案件，请先执行 /init-case"。

## 无参数：查看上下文

展示 `case-context.json` 的核心内容：

```
━━━ 案件上下文 ━━━

**案由：** {case_type}
**原告：** {plaintiff}
**被告：** {defendants 列表}

**案件背景：**
{background}

**律师假设：**
{lawyer_hypotheses 列表}

**策略要点：**
{lawyer_strategy_notes 列表}

**关注焦点：**
{focus_points 列表}

**法律要件覆盖：**
| 要件 | 状态 | 支撑证据 | 缺口说明 |
|------|------|----------|----------|
| {每项一行} |

**动态更新记录：** {updates 条数} 条
{如果有 updates，简要列出}
```

## 带参数：补充信息

律师可以在对话中补充案件信息：
- "被告方新增了一个主体：厚泽后勤服务有限公司"
- "新了解到合同签订时存在胁迫情节"
- "诉讼策略调整为优先主张加速到期"

Agent 将补充信息追加到 `case-context.json` 的 `updates` 数组：

```json
{
  "source": "律师补充",
  "content": "{律师补充的内容}",
  "timestamp": "{当前时间}"
}
```

确认：
```
已记录：{律师补充的内容摘要}
案件上下文已更新，共 {n} 条动态更新记录。
```
